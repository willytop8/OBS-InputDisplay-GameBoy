# GB Input Display — Agent Instructions

## Current Phase: A (OBS Input Overlay Preset)

This is a **single-file HTML prototype** delivered as an OBS Browser Source. It renders Game Boy button inputs using inline SVG + vanilla JS. The future monorepo architecture (TypeScript, Vite, Zod, packages/) is planned but NOT yet built.

## Commands

```bash
# No build step — the prototype is a standalone HTML file
# Open in browser to test: firefox apps/overlay-renderer/index.html
# Package for distribution: bash scripts/package.sh [version]
```

## Project Structure

```
apps/overlay-renderer/index.html   # The browser-source renderer (HTML + inline SVG + vanilla JS)
presets/player-silver/             # Default theme SVG (standalone + synced with inline version)
docs/                              # Installation guide, architecture decisions, research
scripts/package.sh                 # Reproducible zip packaging
releases/                          # Built packages (gitignored)
```

## Architecture (Phase A)

```
Keyboard/Gamepad → Input Overlay Plugin → WebSocket (ws://localhost:16899)
                                                  ↓
                                         index.html (Browser Source)
                                                  ↓
                                         Inline SVG + CSS transitions
                                                  ↓
                                            OBS Studio
```

## Conventions

- Vanilla JS (no build step — OBS Browser Source loads from file://)
- Inline SVG with `data-action` attributes on interactive elements
- CSS custom properties for all theme colors
- `.pressed` class toggled by JS → CSS handles transitions
- Theme SVG class hierarchy: `*-group.btn-base[data-action]` wrappers

## Definition of Done (Phase A)

- All 8 Game Boy actions render with press/release feedback
- Keyboard test mode works without any plugin
- WebSocket client connects to Input Overlay and handles keyboard + gamepad
- Setup guide works from clean OBS profile
- No Nintendo copyrighted assets in any committed file
- Package script produces reproducible zip

## Testing

- Manual: open `index.html` in browser, press Arrow keys + Z/X
- Manual: load as OBS Browser Source with Input Overlay plugin enabled
- Automated tests (Vitest, Playwright) planned for Phase A+ monorepo migration
